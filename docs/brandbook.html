Design System v1.0

Work Dashboard

A minimalist design system built for clarity, focus, and quiet confidence. Inspired by the restraint of Linear, the precision of Apple, and the simplicity of Vercel.

Foundation

Design Principles

Every design decision follows these four principles. They're listed in priority order — when principles conflict, the higher one wins.

Clarity over decoration

The interface should be invisible. Every element earns its place by helping the user understand their data faster. Remove anything that doesn't directly serve comprehension.

Content is the interface

Numbers, labels, and status indicators are the primary visual elements. Chrome and ornamentation exist only to support the hierarchy of data.

Calm confidence

The design should feel premium through restraint, not spectacle. Muted colors, generous whitespace, and subtle transitions. It should never scream.

Density when needed

A dashboard shows a lot of data. We don't sacrifice information density for aesthetics — we find the balance where both work.

Color usage rules
Do Use the accent color only for primary actions and interactive elements (buttons, links, focus rings).
Do Use semantic colors (success/warning/error) only for their intended status meaning.
Don't Use color for decoration or to "make things interesting". Restrain wins.
Don't Use gradients, neon glows, or color overlays. Flat, solid colors only.

Icons

Use Lucide icons (the default for shadcn/ui). Stroke width 1.5px, size 16px for inline and 20px for standalone. Match text color — never use accent color for decorative icons.

Do Use icons to reinforce meaning, not as decoration. Every icon should clarify, not clutter.
Do Keep icons optically consistent — use the same stroke weight and size within a context.
Don't Use filled/solid icons. Outline style only, consistent with the minimal aesthetic.
Don't Use emojis in the product UI. Emojis feel casual and undermine the premium tone.

Motion

Animation should be nearly invisible — it smooths transitions and confirms actions, never entertains. If a user notices the animation, it's too much.

100ms — Hover states, focus rings, color changes. Instant feedback.
200ms — Tooltips, dropdowns, panels sliding in. Perceptible but brisk.
300ms — Page transitions, modal entrances. Slow enough to orient, never sluggish.
Easing: ease — Default easing for all transitions. No bouncing, no spring physics.
No decorative animation — No floating elements, pulse effects, skeleton shimmer beyond initial load, or parallax.

Voice & Tone

How the product speaks. This applies to labels, tooltips, empty states, error messages, and any user-facing text.

We are
  • Calm — no exclamation marks, no urgency theater
  • Direct — say what happened, what to do next
  • Respectful — the user is competent, not a beginner
  • Precise — specific numbers over vague descriptions
We are not
  • Playful — no jokes, no emojis, no whimsy
  • Corporate — no jargon, no stiffness
  • Alarmist — errors are facts, not emergencies
  • Verbose — every word must earn its place

General Rules

A quick reference for making design decisions that stay on-brand.

Do
  • Use whitespace generously — it's a feature, not waste
  • Let typography create hierarchy (size + weight + color)
  • Keep interactions predictable and fast
  • Use borders for separation, shadows for elevation
  • Test both light and dark themes
  • Keep data dense but scannable
Don't
  • Use gradients, glows, or decorative effects
  • Add color "to make things interesting"
  • Use more than one accent color in a single view
  • Add animation that users will notice consciously
  • Use ALL CAPS except in tiny labels (12px)
  • Sacrifice information density for aesthetics
