# UI polish: from skeleton to finished look

Status: active
Created: 2026-04-05
Completed: —

## Purpose

The app currently looks like an unstyled prototype: bare headings floating in
white space, a skeletal sidebar, and sparse stream cards. The brandbook defines
a premium minimalist design system (Linear/Apple-inspired), but the actual
implementation uses mostly default shadcn/ui with no customisation. This plan
brings the existing pages up to the visual standard defined in the brandbook
— without adding new features or data-fetching logic.

After this work, every page of the app will feel intentional and finished: a
clear layout structure, a polished sidebar, proper empty states, and stream
cards that look designed rather than generated.

## Scope

**Included:**
- Global layout shell (max-width, content centering, sidebar refinement)
- Page header pattern (consistent across all pages)
- Empty state component and application to stub pages (Dashboard, Time, Finances)
- Stream list card visual upgrade (avatar, better spacing, richer visual weight)
- Stream detail page layout refinement
- Typography/spacing audit against the brandbook

**Excluded:**
- New features (time logging UI, finance logging UI, dashboard data widgets)
- Backend changes
- Dark theme (deferred — light theme is primary per brandbook)
- Brandbook file changes (the design system is good; the problem is implementation)

## Phases

### Phase 1 — Layout shell and sidebar

**Files to edit:**
- `frontend/src/app/layout.tsx`
- `frontend/src/components/layout/sidebar-nav.tsx`
- `frontend/src/app/globals.css`

**What to do:**

1. **Content area max-width.** Wrap the `{children}` slot inside `<main>` in
   a container div: `max-w-6xl` (1152px). This prevents content from swimming
   on wide screens. The container should NOT be centered with auto margins —
   content should start from the left, naturally bounded.  Use `w-full max-w-6xl`
   on the wrapper.

2. **Main area padding.** Change `<main>` from `p-6` to `px-8 py-6` for a bit
   more horizontal breathing room while keeping vertical compact.

3. **Sidebar visual distinction.** The sidebar and main content both use
   `bg-background` (#FAFAFA). Options to fix:
   - Give the sidebar a slightly different background tint. In `globals.css`,
     add a `--sidebar-bg` variable set to `#FFFFFF` (card background) while the
     main area stays `#FAFAFA`. Apply via `bg-[var(--sidebar-bg)]` or just
     `bg-card` on the `<aside>`.
   - Alternatively, flip it: sidebar stays #FAFAFA (current), main area gets
     `bg-card` (#FFFFFF). This creates a subtle but real contrast.
   - **Decision:** Use the flip approach. Keep sidebar as `bg-background`
     (#FAFAFA), change `<main>` background to `bg-card` (#FFFFFF). This makes
     the content area feel like an elevated surface, matching the brandbook's
     "bg-secondary = #FFFFFF for cards/content".

4. **Sidebar bottom section.** Add a visual separator and a version label at
   the bottom of the sidebar. Structure:

   ```
   <aside>  (h-full flex flex-col)
     <div>  title  </div>
     <nav>  links  </nav>
     <div class="mt-auto">  separator + "v1.0" label  </div>
   </aside>
   ```

   Use `<Separator />` component (already exists). The version text: `text-xs
   text-muted-foreground`. This gives the sidebar a "finished" bottom.

5. **Sidebar active state refinement.** Current active state is
   `bg-accent text-accent-foreground`. This maps to `#F4F4F5` — very subtle.
   Add a left accent indicator: a 2px left border in `border-primary` color
   on the active item. Use `border-l-2 border-primary` on active items,
   remove `rounded-md` left rounding with `rounded-r-md rounded-l-none`.
   This adds a Linear-style active indicator.

6. **Sidebar nav spacing.** Increase gap from `gap-0.5` (2px) to `gap-1`
   (4px) between nav items for better scannability.

**Testable result:** Open the app at 1440px width. The content no longer
stretches edge-to-edge. The sidebar has a visible bottom with version info.
There is a subtle background difference between sidebar and content area.
The active nav item has a left accent border.

### Phase 2 — Page header component and empty states

**Files to create:**
- `frontend/src/components/layout/page-header.tsx`
- `frontend/src/components/layout/empty-state.tsx`

**Files to edit:**
- `frontend/src/app/page.tsx` (Dashboard)
- `frontend/src/app/time/page.tsx`
- `frontend/src/app/finances/page.tsx`
- `frontend/src/app/streams/page.tsx`
- `frontend/src/app/streams/[id]/page.tsx`
- `frontend/src/i18n/ru.ts`

**What to do:**

1. **PageHeader component.** A reusable header for every page. Props:
   `title: string`, `subtitle?: string`, `action?: React.ReactNode`.
   Layout: title and subtitle on the left, action button on the right.
   Title: `text-xl font-semibold tracking-tight` (not `text-2xl` — slightly
   smaller for a dashboard feel). Subtitle: `text-sm text-muted-foreground
   mt-0.5`. Action: right-aligned. The whole header has `mb-6` bottom margin
   as a separator from content.

   ```tsx
   <div className="flex items-start justify-between">
     <div>
       <h1 className="text-xl font-semibold tracking-tight">{title}</h1>
       {subtitle && <p className="mt-0.5 text-sm text-muted-foreground">{subtitle}</p>}
     </div>
     {action && <div>{action}</div>}
   </div>
   ```

2. **EmptyState component.** A centered block for pages without content.
   Matches the brandbook's `.empty-state` pattern. Props:
   `icon: LucideIcon`, `title: string`, `description: string`,
   `action?: React.ReactNode`.

   Structure:
   ```tsx
   <div className="flex flex-col items-center justify-center py-16 text-center">
     <div className="mb-4 flex h-11 w-11 items-center justify-center rounded-lg bg-muted">
       <Icon className="h-5 w-5 text-muted-foreground" strokeWidth={1.5} />
     </div>
     <h2 className="text-base font-semibold">{title}</h2>
     <p className="mt-1.5 max-w-[280px] text-sm text-muted-foreground">{description}</p>
     {action && <div className="mt-5">{action}</div>}
   </div>
   ```

3. **Apply PageHeader** to all pages: Dashboard, Streams list, Stream detail
   (back link above it), Time, Finances. Replace the ad-hoc header markup in
   each page with `<PageHeader>`.

4. **Apply EmptyState** to stub pages:
   - **Dashboard:** icon=`LayoutDashboard`, title=`ru.dashboard.empty.title`,
     description=`ru.dashboard.empty.description`. No action button.
   - **Time:** icon=`Clock`, title=`ru.time.empty.title`,
     description=`ru.time.empty.description`. No action button (feature not
     built yet).
   - **Finances:** icon=`Wallet`, title=`ru.finances.empty.title`,
     description=`ru.finances.empty.description`. No action button.

5. **Add i18n keys** in `ru.ts`:
   ```
   dashboard.empty.title = "Данных пока нет"
   dashboard.empty.description = "Здесь появится сводка, когда будут добавлены потоки работы и записи."
   time.empty.title = "Записей пока нет"
   time.empty.description = "Здесь появится учёт времени по потокам работы."
   finances.empty.title = "Записей пока нет"
   finances.empty.description = "Здесь появятся доходы и расходы по потокам работы."
   ```

6. **Streams empty state** already exists in `streams/page.tsx` — refactor it
   to use the new `EmptyState` component for consistency.

**Testable result:** Every page has a consistent header pattern. Stub pages
show a calm, centered empty state with an icon and description instead of bare
headings in a void.

### Phase 3 — Stream cards and detail page

**Files to edit:**
- `frontend/src/app/streams/page.tsx`
- `frontend/src/app/streams/[id]/page.tsx`

**What to do:**

1. **Stream card upgrade.** Each stream card in the grid should feel denser
   and more intentional. Changes to the card content:

   - Add an avatar circle (initials from stream name, first 2 chars
     uppercased). Use a `div` with `h-9 w-9 rounded-full bg-muted flex
     items-center justify-center text-xs font-semibold text-muted-foreground`.
   - Move the avatar to the CardHeader, alongside the title. Use flexbox:
     avatar left, title + subtitle right.
   - CardHeader layout: `flex flex-row items-center gap-3` (remove the
     default grid layout). Title stays `text-base font-medium`.
     Below the title, show the type as plain text `text-xs text-muted-foreground`
     instead of a badge, to reduce badge clutter.
   - CardContent: just the status badge. If description exists, show it
     below the badge as `text-sm text-muted-foreground line-clamp-2`.
   - Hover state: keep the existing `hover:border-zinc-300 hover:shadow-md`,
     it's fine.

2. **Grid adjustment.** Change `lg:grid-cols-3` to `sm:grid-cols-2 lg:grid-cols-3`
   to ensure cards have reasonable width. With the new max-width from Phase 1,
   three columns in 1152px max = ~370px each, which is fine.

3. **Stream detail page layout.** The Card currently stretches full-width. Add
   `max-w-2xl` (672px) to constrain the detail card to a comfortable reading
   width. This prevents the card from looking like a thin strip across a wide
   screen.

4. **Detail page metadata line.** The creation date line is fine. Ensure
   consistent spacing: use `Separator` between the info section and the
   action buttons.

**Testable result:** Stream cards show an avatar, name, type text, and status
badge. Cards feel denser and more "designed". The detail page card is
comfortably constrained to ~672px width.

### Phase 4 — Typography and spacing audit

**Files to edit:**
- `frontend/src/app/globals.css`
- `frontend/src/components/ui/card.tsx` (possibly)

**What to do:**

1. **Font size audit.** The brandbook specifies `14px` as the default body
   size (`--text-base`). Tailwind's default `text-sm` is `0.875rem` (14px) —
   this is correct. Verify that all body text in the app uses `text-sm` or
   `text-base` (Tailwind's `text-base` is 16px, which is the brandbook's
   `--text-md`). The Card component sets `text-sm` on the card root — good.

2. **Spacing validation.** Walk through key spacing values:
   - Card internal padding: shadcn uses `px-4 py-4` (16px). Brandbook
     recommends `--space-5` (20px) for card body. Adjust Card's `py-4` to
     `py-5` and keep `px-4` — or adjust both to `p-5`. Decision: change Card
     root from `py-4` to `py-5` for slightly more breathing room inside cards.
   - Page header bottom margin: `mb-6` (24px) — matches brandbook's
     `--space-6` for section padding.
   - Grid gap: `gap-4` (16px) — matches brandbook's `--space-4`. Fine.

3. **Border radius check.** The Card component uses `rounded-xl` (12px).
   The brandbook defines `--radius-lg: 12px` for cards. Match confirmed.

4. **Shadow check.** Card uses `shadow-xs`. The brandbook defines barely-there
   shadows. For the main content cards (streams), consider upgrading to
   `shadow-sm` for slightly more elevation against the now-white main
   background. Apply via the Card usage in streams page, not the Card
   component itself (to keep the component flexible).

5. **Global body font-size.** Ensure `html` or `body` has the correct base
   font size. Currently `html` has `font-sans` applied via `@layer base`.
   Add `text-sm` to body in globals.css `@layer base` to set the global
   default to 14px (matching brandbook's 14px body default).

**Testable result:** Open DevTools, inspect card padding — should be 20px
vertical. Body font size is 14px. Cards have subtle shadow elevation.
Typography feels tight and intentional, not default-Tailwind loose.

## Decision log

- Decision: main area gets `bg-card` (#FFFFFF), sidebar stays `bg-background` (#FAFAFA)
  Why: creates subtle surface contrast without adding new CSS variables; matches brandbook's "bg-secondary for cards/content" pattern
  Date: 2026-04-05

- Decision: page title uses `text-xl` (20px) not `text-2xl` (24px)
  Why: 24px is too large for a dashboard page title in a sidebar layout — it dominates instead of anchoring; 20px matches the brandbook's `--text-xl` for section headings
  Date: 2026-04-05

- Decision: stream type shown as plain text, not badge, inside cards
  Why: two badges per card (type + status) is visually noisy; type is categorization (plain text), status is state (badge)
  Date: 2026-04-05

- Decision: no changes to the brandbook itself
  Why: the design system is well-defined; the gap is in implementation, not specification
  Date: 2026-04-05
